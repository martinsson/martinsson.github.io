Test et refactoring de legacy

Améliorer (refactorer) du code existant sans risque de régression à l'aide d'écriture de tests unitaires, de l'utilisation de la couverture des tests et les refactorings automatisés de nos IDEs

Bénéfices pour les participants

    Voir comment rendre du code non testable testable. Connaître des principes pour bien refactorer. Apprendre comment les tests nous montrent des problèmes dans notre design. Être surpris par la capacité de refactoring automatique de nos IDEs.
